A brand-neutral starter template that hands your AI coding assistant a curated toolkit — plugins, skills, and design-system templates — plus a runbook it follows to set everything up for you. You bring the idea; the agent wires up the workshop. 01What this starter isApa itu starter ini

Think of it as a fully-stocked workshop that arrives flat-packed. Every drawer is labelled but empty until you open it. When you first load the project, the AI agent reads a setup runbook and — with your approval at each step — installs the tools, verifies them, asks about your brand, and fills in the project guides. Anggap saja seperti bengkel lengkap yang datang dalam kondisi flat-pack. Setiap laci sudah diberi label tapi kosong sampai kamu membukanya. Saat pertama kali membuka proyek, agen AI membaca runbook setup dan — dengan persetujuanmu di tiap langkah — memasang alat-alatnya, memverifikasinya, menanyakan brand kamu, dan mengisi panduan proyek.

🧩 Curated capabilitiesKemampuan terkurasi

A hand-picked .claude/ folder of plugins (commands + specialist agents), skills (always-on know-how), and design/brand templates — chosen to cover the common jobs of building software. Folder .claude/ pilihan berisi plugin (perintah + agen spesialis), skill (pengetahuan yang selalu aktif), dan template design/brand — dipilih untuk menutup pekerjaan umum dalam membangun software.

📋 A setup runbookRunbook setup

setup.md is a phased checklist the agent follows: health-check, install with your approval, verify, capture your brand, then initialize the guides. Re-running it is safe — it tracks what's already done. setup.md adalah checklist bertahap yang diikuti agen: cek kondisi, pasang dengan persetujuanmu, verifikasi, rekam brand kamu, lalu inisialisasi panduannya. Menjalankannya ulang aman — ia melacak apa yang sudah selesai.

🎨 Brand-neutral by defaultNetral-merek secara default

Ships with no colours, fonts, or company name baked in. The setup interview fills those placeholders with your brand, so everything the agent builds looks like you. Datang tanpa warna, font, atau nama perusahaan yang tertanam. Wawancara setup mengisi placeholder itu dengan brand kamu, jadi semua yang dibangun agen terlihat seperti milikmu.

🙋 Built for non-codersDibuat untuk non-coder

You don't install anything by hand. You describe what you want; the agent does the plumbing and explains each step in plain language before acting. Kamu tidak memasang apa pun secara manual. Kamu cukup menjelaskan keinginanmu; agen yang mengurus teknisnya dan menjelaskan tiap langkah dengan bahasa sederhana sebelum bertindak.

Source of truth: the inventory below is drawn from .claude/setup.config.json — the authoritative manifest — and each plugin's own plugin.json. If the config changes, that file wins. Sumber kebenaran: daftar di bawah diambil dari .claude/setup.config.json — manifes resmi — dan plugin.json milik tiap plugin. Kalau config berubah, file itulah yang menang.

02Getting startedMulai

On first load, if setup hasn't run yet (state.setupCompleted is false), just ask the agent to "follow setup.md." It walks these phases, pausing for your OK before anything installs. Saat pertama dibuka, kalau setup belum jalan (state.setupCompleted bernilai false), cukup minta agen untuk "ikuti setup.md." Ia akan menelusuri tahap-tahap ini, berhenti menunggu persetujuanmu sebelum memasang apa pun.

Health checkCek kondisi

The agent inspects what's already installed (claude plugin list) so it never double-installs and knows exactly what's missing. Agen memeriksa apa yang sudah terpasang (claude plugin list) supaya tidak memasang dua kali dan tahu persis apa yang kurang.

Install plugins — with your approvalPasang plugin — dengan persetujuanmu

Adds the marketplaces and installs each required plugin (claude plugin install <name>@<marketplace>). Optional external token tools are offered separately, installed only if you opt in. Menambahkan marketplace dan memasang tiap plugin wajib (claude plugin install <name>@<marketplace>). Alat token eksternal opsional ditawarkan terpisah, dipasang hanya kalau kamu setuju.

VerifyVerifikasi

Confirms skills and output styles auto-loaded once the workspace is trusted (a /reload-plugins refresh), so the toolkit is actually live. Memastikan skill dan gaya output sudah auto-load setelah workspace dipercaya (refresh /reload-plugins), jadi perangkatnya benar-benar aktif.

Brand interviewWawancara brand

Asks about your company name, colours, fonts, and tone, then writes them into the brand skill and design tokens — flipping brandConfigured to true. Menanyakan nama perusahaan, warna, font, dan tone kamu, lalu menuliskannya ke skill brand dan design token — mengubah brandConfigured jadi true.

Initialize the guidesInisialisasi panduan

Fills the <PLACEHOLDER> fields in CLAUDE.md / AGENTS.md so every agent that opens the project reads the same, accurate guide. Mengisi kolom <PLACEHOLDER> di CLAUDE.md / AGENTS.md supaya setiap agen yang membuka proyek membaca panduan yang sama dan akurat.

How things install: plugins come from a bundled local marketplace (vibe-code-starter, works offline) plus two external GitHub marketplaces (opensesh/KARIMO, DietrichGebert/ponytail). Skills and output styles need no install — they auto-load once the workspace is trusted. Cara pemasangannya: plugin berasal dari marketplace lokal bawaan (vibe-code-starter, jalan offline) plus dua marketplace GitHub eksternal (opensesh/KARIMO, DietrichGebert/ponytail). Skill dan gaya output tidak perlu dipasang — semuanya auto-load begitu workspace dipercaya.

03Your vibe-coding workflowAlur kerja vibe-coding kamu

Setup happens once — then the work begins. There isn't one workflow, though: there's a different one for each kind of job. Pick a tab below. Setup cuma sekali — lalu pekerjaannya dimulai. Tapi alur kerjanya bukan cuma satu: ada yang berbeda untuk tiap jenis pekerjaan. Pilih tab di bawah.

For most work — features, fixes, tweaks. You drive small loops: describe it, the agent builds it, you ship it, repeat. You mostly speak plain language; the badges show which skill switches on automatically and which command you type at each stage. Untuk sebagian besar pekerjaan — fitur, perbaikan, penyesuaian. Kamu menjalankan loop kecil: jelaskan, agen membangunnya, kamu rilis, ulangi. Kamu mayoritas bicara dengan bahasa biasa; badge menunjukkan skill mana yang aktif otomatis dan perintah mana yang kamu ketik di tiap tahap.

08Optional token-saving toolsAlat hemat token opsional

These are external command-line tools, not Claude Code plugins. They cut how many tokens the agent burns — handy on long sessions or large codebases. The setup runbook offers each and installs only if you opt in. Ini alat command-line eksternal, bukan plugin Claude Code. Keduanya memangkas jumlah token yang dipakai agen — berguna untuk sesi panjang atau codebase besar. Runbook setup menawarkan masing-masing dan memasang hanya kalau kamu setuju.

rtk AvailableTersedia

"Rust Token Killer" — compresses command output"Rust Token Killer" — memampatkan output perintah

Sits between the agent and your tools (ls, cat, grep, git, test runners) and trims their output before the model reads it — 60–90% fewer tokens from tool output, under 10 ms overhead. Duduk di antara agen dan alat-alatmu (ls, cat, grep, git, test runner) dan memangkas output-nya sebelum dibaca model — 60–90% lebih sedikit token dari output alat, overhead di bawah 10 ms.

Integrates via: a hook (rtk hook claude) · Repo: rtk-ai/rtk
